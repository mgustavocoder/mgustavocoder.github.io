Cloud Design Patterns - Design e implementação

  • Ambassador
    • Crie serviços auxiliares que enviem solicitações de rede em nome de um serviço ou aplicativo do consumidor.
  • Anti-Corruption Layer
    • Implemente uma fachada ou camada adaptadora entre um aplicativo moderno e um sistema legado.
  • Backends for Frontends
    • Crie serviços de back-end separados para serem consumidos por aplicativos ou interfaces de front-end específicos.
  • CQRS
    • Separe operações que leem dados de operações que atualizam dados usando interfaces separadas.
  • Compute Resource Consolidation
    • Consolide várias tarefas ou operações em uma única unidade computacional.
  • External Configuration Store
    • Mova as informações de configuração do pacote de implantação do aplicativo para um local centralizado.
  • Gateway Aggregation
    • Use um gateway para agregar várias solicitações individuais em uma única solicitação.
  • Gateway Offloading
    • Descarregue a funcionalidade de serviço compartilhado ou especializado para um proxy de gateway.
  • Gateway Routing
    • Encaminhe solicitações para vários serviços usando um único endpoint.
  • Leader Election
    • Coordene as ações executadas por uma coleção de instâncias de tarefas colaborativas em um aplicativo distribuído, elegendo uma instância como líder que assume a responsabilidade de gerenciar as outras instâncias.
  • Pipes and Filters
    • Divida uma tarefa que executa processamento complexo em uma série de elementos separados que podem ser reutilizados.
  • Sidecar
    • Implante componentes de um aplicativo em um processo ou contêiner separado para fornecer isolamento e encapsulamento.
  • Static Content Hosting
    • Implante conteúdo estático em um serviço de armazenamento baseado em nuvem que possa entregá-lo diretamente ao cliente.
  • Strangler Fig
    • Migre de forma incremental um sistema legado substituindo gradualmente partes específicas de funcionalidade por novos aplicativos e serviços.
