Princípios de Design Arquiteturais.

  • Design for self-healing.

    • Em um sistema distribuído, as falhas acontecem. Projete seu aplicativo para ser autocorretivo quando ocorrerem falhas.

  • Make all things redundant.

    • Crie redundância em seu aplicativo para evitar pontos únicos de falha.

  • Minimize coordination.

    • Minimize a coordenação entre os serviços de aplicativos para obter escalabilidade.

  • Design to scale out.

    • Projete seu aplicativo para que ele possa ser dimensionado horizontalmente, adicionando ou removendo novas instâncias conforme a demanda exigir.

  • Partition around limits.

    • Use o particionamento para contornar limites de banco de dados, rede e computação.

  • Design for operations.

    • Projete seu aplicativo para que a equipe de operações tenha as ferramentas necessárias.

  • Use managed services.

    • Quando possível, use plataforma como serviço (PaaS) em vez de infraestrutura como serviço (IaaS).

  • Use an identity service.

    • Use uma plataforma de identidade como serviço (IDaaS) em vez de criar ou operar a sua própria.

  • Use the best data store for the job.

    • Escolha a tecnologia de armazenamento mais adequada para seus dados e como ela será usada.

  • Design for evolution.

    • Todos os aplicativos bem-sucedidos mudam com o tempo. Um design evolutivo é a chave para a inovação contínua.

  • Build for the needs of business.

    • Cada decisão de projeto deve ser justificada por um requisito de negócios.
