Princípios FIRST para testes de unidade.

Sim, são testes de unidade, não testes unitários, e os princípios FIRST são:

  • Fast - Devem ser rápidos, pois se for devagar, não serão executados.
  • Isolated - Um teste não deve depender ou interferir em outro teste.
  • Repeatable - Testes devem ser repetíveis, por exemplo, não deve depender de uma data que possa mudar e quebrar depois de um tempo.
  • Self-verifying - O teste deve se auto verificar, para saber se deu erro. Não se deve precisar de uma pessoa que valide o log dos testes de unidade para verificar se os testes quebraram ou não.
  • Timely - Essa regra diz que os testes devem ser escritos antes da escrita do código.
