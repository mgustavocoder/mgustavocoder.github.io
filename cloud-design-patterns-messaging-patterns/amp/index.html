Cloud Design Patterns - Mensageria

Matheus Gustavo - 21 Jul 2022

  • Asynchronous Request-Reply

    • Separe o processamento de back-end de um host de front-end, onde o processamento de back-end precisa ser assíncrono, mas o front-end ainda precisa de uma resposta clara.

  • Claim Check

    • Divida uma mensagem grande em uma verificação de solicitação e uma carga útil para evitar sobrecarregar um barramento de mensagens.

  • Choreography

    • Faça com que cada componente do sistema participe do processo de tomada de decisão sobre o fluxo de trabalho de uma transação comercial, em vez de depender de um ponto de controle central.

  • Competing Consumers

    • Permita que vários consumidores simultâneos processem mensagens recebidas no mesmo canal de mensagens.

  • Pipes and Filters

    • Divida uma tarefa que executa processamento complexo em uma série de elementos separados que podem ser reutilizados.

  • Priority Queue

    • Priorize as solicitações enviadas aos serviços para que as solicitações com prioridade mais alta sejam recebidas e processadas mais rapidamente do que aquelas com prioridade mais baixa.

  • Publisher-Subscriber

    • Habilite um aplicativo para anunciar eventos para vários consumidores interessados de forma assíncrona, sem acoplar os remetentes aos destinatários.

  • Queue-Based Load Leveling

    • Use uma fila que atue como um buffer entre uma tarefa e um serviço que ela chama para suavizar cargas pesadas intermitentes.

  • Scheduler Agent Supervisor

    • Coordene um conjunto de ações em um conjunto distribuído de serviços e outros recursos remotos.

  • Sequential Convoy

    • Processe um conjunto de mensagens relacionadas em uma ordem definida, sem bloquear o processamento de outros grupos de mensagens.
